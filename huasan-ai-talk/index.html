02 · Prompt

Prompt 是 LLM 的方向盘，不是过时技巧。

模型越强，prompt 越像方向盘——发散时决定 AI 往哪里想，收敛时决定按什么标准交付。

LLM 读你的输入，
不只是文字 ——
而是一组信号。
任务信号 上下文信号 专业度信号 质量标准信号
↓ 共同塑造 LLM 的输出空间
写法 A · 升维

发散型 Prompt

目标：打开空间
  • 关键词、对比关系、反常识问题
  • 把 AI 推进更高价值的思考空间
  • 不急着规定格式，不急着控制步骤
适用 · 选题 / 洞察 / 产品方向 / 商业机会
⇄vs
写法 B · 约束

收敛型 Prompt

目标：稳定交付
  • 角色 / 北极星目标 / 证据等级 / 输出结构
  • 把"好洞见"从感觉变成可检查条件
  • 真正有价值的是质量门槛，不是字数
适用 · 分析报告 / 知识卡片 / 结构化产出
1 姜胡说案例 · 认知悖论

"认知悖论"让 AI 去找矛盾结构

在[个人成长 / 赚钱致富 / 自媒体]领域，普遍存在的认知悖论是什么——即那些人们深信不疑，却在根本上阻碍他们成功的想法。
三个关键词都在升维：认知悖论找矛盾、深信不疑找共识盲点、阻碍成功找高影响因子。
SOURCE · 姜胡说小报童
2 姜胡说案例 · 核心数字资产

把产品思考从"市场有什么"换成"我的不对称优势是什么"

请你找到对我来说已经轻车熟路，但是对其他人难如登天的"核心数字资产"，稍加更改对他们来说就特别有用的"产品"。
"轻车熟路 vs 难如登天"构成价值差——这比"我能做什么产品"好用得多。

03 · Skills

Skills 把反复有效的经验，沉淀成能力包。

Prompt 解决一次对话。Skill 解决一类任务的复用——一个目录、一份 SKILL.md、按需加载的 references / scripts / assets。

Tier 0 · 元数据
Always loaded
name · description · 触发条件
何时加载 · AI 启动时全部读入，决定要不要往下展开
Tier 1 · 主文档
On match
SKILL.md · 完整工作流、禁止事项、检查点
何时加载 · 用户请求和 description 匹配时
Tier 2 · 资源
On demand
references/*.md · 细节资料
scripts/*.{sh,py,js} · 可执行脚本
assets/* · 模板与示例
何时加载 · SKILL.md 里 reference 到对应文件时
旧结构 · 反例

大而全 skill

获取灵感 → 选题 → 画伞风格 → 手动改写 → 反向优化 → 生成配图

想单独用生图能力时，要从大 skill 里拆。agent 读了很多无关上下文，完整工作流会带偏单点任务——功能越多，边界越模糊。

新结构 · 像乐高

原子化 skills

analyze → outline → draft → polish → ill

skill 的价值来自清晰边界。边界越清楚，越容易单独使用，也越容易组合。

文件系统让 skill 有地方放，scripts 让 skill 不只是文字说明，测试 + Git + 权限确认让 agent 能闭环验证——这就是 skill 从"提示模板"升级成"能力包"的物理基础。

学 AI 的一个习惯
把缩写还原成英文

很多 AI 工具、命令行概念都是缩写。还原成完整英文，能快速理解它到底在做什么——也帮你看见工具背后的设计意图。

cdchange directory
每天看见、每天用——还原成英文以后就很直观：切换目录。
bashBourne Again Shell
Stephen Bourne 写了 sh；Brian Fox 重写增强，叫 Bash——既致敬 Bourne，也借英文 Born Again 的双关。

04 · Harness Engineering · 重头戏

Harness 把 agent teams 变成可控系统。

Prompt 管单次质量。Skill 管类型复用。Harness 管长任务可靠性——一套工程结构，让 AI 在长任务里持续、稳定、可验证地工作。

06 · 最终收束

AI 时代最终需要的，是人的判断力。

AI 能力会继续变强。执行会越来越便宜。普通人要同时练四件事——而最难也最稀缺的，是最后一件。

最终判断

Prompt 训练表达，
Skill 训练流程，
Harness 训练系统，
Founder OS 训练判断。

AI 时代最大的风险，是 build 太容易以后，
人更容易逃避"这件事到底值不值得做"。
